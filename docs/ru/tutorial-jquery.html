DOM Manipulation

Еще один класс функций, который часто считается трудным для тестирования это код, который напрямую манипулирует DOM. Давайте взглянем на то, как мы можем протестировать следующий фрагмент кода jQuery, который прослушивает событие click, асинхронно загружает данные и задает содержимое для span.

// displayUser.js
'use strict';

const $ = require('jquery');
const fetchCurrentUser = require('./fetchCurrentUser.js');

$('#button').click(() => {
  fetchCurrentUser(user => {
    const loggedText = 'Logged ' + (user.loggedIn ? 'In' : 'Out');
    $('#username').text(user.fullName + ' - ' + loggedText);
  });
});

Снова создадим тестовый файл в папке __tests__ /:

// __tests__/displayUser-test.js
'use strict';

jest.mock('../fetchCurrentUser');

test('displays a user after a click', () => {
  // Set up our document body
  document.body.innerHTML =
    '<div>' +
    '  <span id="username" />' +
    '  <button id="button" />' +
    '</div>';

  // This module has a side-effect
  require('../displayUser');

  const $ = require('jquery');
  const fetchCurrentUser = require('../fetchCurrentUser');

  // Tell the fetchCurrentUser mock function to automatically invoke
  // its callback with some data
  fetchCurrentUser.mockImplementation(cb =>  {
    cb({
      fullName: 'Johnny Cash',
      loggedIn: true,
    });
  });

  // Use jquery to emulate a click on our button
  $('#button').click();

  // Assert that the fetchCurrentUser function was called, and that the
  // #username span's inner text was updated as we'd expect it to.
  expect(fetchCurrentUser).toBeCalled();
  expect($('#username').text()).toEqual('Johnny Cash - Logged In');
});

Тестируемая функция добавляет прослушиватель событий для элемента DOM #button, поэтому нам необходимо правильно настроить DOM для тестирования. Jest поставляется с библиотекой jsdom, которая имитирует DOM окружение, как если бы вы использовали браузер. Это означает, что мы можем наблюдать за любым вызовом DOM API, так же как это происходило бы в браузере!

Мы имитируем fetchCurrentUser.js, чтобы наш тест не совершал реальных запросов к сети и вместо этого использовал локальные имитированные данные. Это гарантирует, что наш тест может выполниться за миллисекунды, а не секунды, и это также гарантирует быстрые итерации юнит тестов.

Код для этого примера доступен в examples/jquery.
