DOM Manipulation

Інший клас функцій, який часто вважається важким для тестування - це код, що прямо взаємодіє з DOM. Давайте подивимося, як ми можемо протестувати наступний фрагмент коду jQuery, що слухає click подію, отримує деякі дані асинхронно і змінює контент span.

// displayUser.js
'use strict';

const $ = require('jquery');
const fetchCurrentUser = require('./fetchCurrentUser.js');

$('#button').click(() => {
  fetchCurrentUser(user => {
    const loggedText = 'Logged ' + (user.loggedIn ? 'In' : 'Out');
    $('#username').text(user.fullName + ' - ' + loggedText);
  });
});

Знову ж таки, ми створюємо тестовий файл в __tests__ / папці:

// __tests__/displayUser-test.js
'use strict';

jest.mock('../fetchCurrentUser');

test('displays a user after a click', () => {
  // Налаштуйте тіло вашого документа
  document.body.innerHTML =
    '<div>' +
    '  <span id="username" />' +
    '  <button id="button" />' +
    '</div>';

  // Цей модуль має побічний ефект
  require('../displayUser');

  const $ = require('jquery');
  const fetchCurrentUser = require('../fetchCurrentUser');

  // Скажіть вашій fetchCurrentUser мок функції автоматично запускати
  // її колбек з деякими даними
  fetchCurrentUser.mockImplementation(cb =>  {
    cb({
      fullName: 'Johnny Cash',
      loggedIn: true,
    });
  });

  // Використайте jquery для симуляції кліку по кнопці
  $('#button').click();

  // Підтвердіть що fetchCurrentUser функція була викликана, і що 
  // внутрішній текст в #username span's був оновлений як ми того очікували.
  expect(fetchCurrentUser).toBeCalled();
  expect($('#username').text()).toEqual('Johnny Cash - Logged In');
});

Функція, що тестується підписується на події DOM елемента #button, тому нам потрібно правильно налаштувати наш DOM для тесту. До складу Jest входить jsdom, який емулює DOM середовище так, ніби ваш код виконується в браузері. Це означає, що кожен DOM API, який ми викливаємо, можна спостерігати так само, як за ним можна спостерігати у браузері!

Ми створюємо мок для fetchCurrentUser.js, тому наш тест не робить справжніх мережевих запитів, а отримує імітацію даних локально. Це гарантує, що наш тест можна завершити за декілька мілісекундах, а не секунд і гарантує високу швидкість інтерації юніт-тестів.

Код цього прикладу доступний в Приклади/jquery.
