DOM Manipulation

Otra clase de funciones que a menudo se considera difícil de probar es el código que manipula directamente el DOM. Vamos a ver cómo podemos probar el siguiente fragmento de código de jQuery que escucha un evento click, trae algunos datos de forma asincrónica y establece el contenido de un elemento span.

// displayUser.js
'use strict';

const $ = require('jquery');
const fetchCurrentUser = require('./fetchCurrentUser.js');

$('#button').click(() => {
  fetchCurrentUser(user => {
    const loggedText = 'Logged ' + (user.loggedIn ? 'In' : 'Out');
    $('#username').text(user.fullName + ' - ' + loggedText);
  });
});

Una vez más, creamos un archivo de test en la capeta __tests__ /:

// __tests__/displayUser-test.js
'use strict';

jest.mock('../fetchCurrentUser');

test('displays a user after a click', () => {
  // Set up our document body
  document.body.innerHTML =
    '<div>' +
    '  <span id="username" />' +
    '  <button id="button" />' +
    '</div>';

  // This module has a side-effect
  require('../displayUser');

  const $ = require('jquery');
  const fetchCurrentUser = require('../fetchCurrentUser');

  // Tell the fetchCurrentUser mock function to automatically invoke
  // its callback with some data
  fetchCurrentUser.mockImplementation(cb =>  {
    cb({
      fullName: 'Johnny Cash',
      loggedIn: true,
    });
  });

  // Use jquery to emulate a click on our button
  $('#button').click();

  // Assert that the fetchCurrentUser function was called, and that the
  // #username span's inner text was updated as we'd expect it to.
  expect(fetchCurrentUser).toBeCalled();
  expect($('#username').text()).toEqual('Johnny Cash - Logged In');
});

La función testeada añade un detector de eventos en el elemento de DOM #button, así que tenemos que configurar nuestro DOM correctamente para la prueba. Jest viene con jsdom que simula un entorno de DOM como si estuvieras en el navegador. Esto significa que se puede observar cada API del DOM que llamamos de la misma manera se observaría en un navegador!

Estamos mockeando fetchCurrentUser.js para que nuestro test no haga una solicitud de red real pero en su lugar mockeamos los datos a nivel local. Esto garantiza que nuestro test puede terminar en milisegundos en lugar de segundos y garantiza una rápida interacción en cada unidad de test.

El código de este ejemplo está disponible en: examples/jquery.
